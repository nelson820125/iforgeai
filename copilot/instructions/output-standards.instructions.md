## Global Output Standards

### Prohibited Content

**Openers** (delete directly, no explanation):
- "Sure, I'll help you..."
- "Of course, here is..."
- "As a [role], I..."
- "Understood, based on your requirements..."
- "Great question..."

**Redundant section headers** (present content directly, no header needed):
- "In summary"
- "It is worth noting that"
- "Taking everything into consideration"
- "From the perspective of XX"
- "It is recommended to consider"

**Vague phrasing** (replace with specific descriptions):
- "There may be an issue" → state what the specific issue is
- "Suggest paying attention to" → state what to pay attention to and why
- "Arrange reasonably" → state the specific arrangement
- "Configure flexibly" → state the specific configuration options

### Must Do

1. **Start with the conclusion**: The first sentence is the most important information — not background framing
2. **Every judgement cites a source**: Reference specific file paths, standard items, or data — do not generalise from "experience"
3. **Ask directly when uncertain**: If ambiguous, raise a specific question — do not assume and then produce a large output
4. **Numbers must be specific**: Do not write "relatively high performance" — write "response time < 200ms"
5. **Do not repeat known context**: Do not restate what the user already said — proceed directly

### Document Writing Standards

- Maximum 3 heading levels (H1 / H2 / H3)
- If a list has more than 5 items, consider grouping with sub-headings
- Do not use exclamation marks (!) to create urgency or enthusiasm
- Tables over long lists, long lists over long paragraphs
- Code blocks must specify language (```csharp, ```typescript, ```markdown, etc.)
